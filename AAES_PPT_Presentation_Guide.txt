AAES - Automated Assignment Evaluation System - Presentation Content (Detailed)

Slide 1: Abstract
• AI-powered platform to automate academic assignment evaluation.
• Supports Theory (OCR) and Programming (Python) submissions.
• Role-based dashboards for Students, Staff, HODs, and Principals.

Slide 2: Problem Statements
• Manual Overhead: Time-consuming manual grading for faculty.
• Inconsistency: Subjective and varied human evaluation results.
• Delayed Feedback: Students wait too long for performance results.
• Lack of Tracking: No real-time institutional monitoring for HOD/Principal.

Slide 3: Literature Survey (Simplified)
• Traditional LMS: No automated subjective grading.
• AI in Education: OCR and LLMs offer new possibilities for evaluation.

Slide 4: Existing System
• Manual Processes: Physical submissions and offline evaluation.
• Offline Records: Records stored in physical registers or Excel.

Slide 5: Existing System Disadvantages
• High Turnaround Time for feedback.
• Potential for Human Bias in marking.
• Administrative Burden on staff for data management.

Slide 6: Proposed System
• Intelligent Evaluation using OCR, LLMs, and Safe Code Execution.
• Unified Role-Based Portals for all stakeholders.
• Real-time Analytics and Governance Dashboards.

Slide 7: Proposed System Advantages
• Instant Feedback loop for students.
• Higher Accuracy and consistency in grading.
• Automated Governance via Risk Intelligence.

Slide 8: Staff & Subject Management
• Assignment Creation: Configure Theory and Coding tasks easily.
• Intelligent Evaluation: AI-assisted grading with manual overrides.
• Subject Analytics: Real-time attendance summary and grade distribution.
• Unified Dashboards: Manage all subjects from a single interface.

Slide 9: Advisor (Mentor) & CCM Features
[Mentor / Advisor Options]
• Student List Management: Detailed view of all mentored students.
• Advisor Attendance: Monitor consistent presence across all subjects.
• Advisor Notes: Log student behavioral and academic progress.
• Performance Monitoring: Cross-subject grade tracking for individuals.
[CCM (Class Committee Meeting)]
• Staff CCM Portal: Register and track meeting minutes and student feedback.
• Action Item Tracking: Ensure student grievances are addressed promptly.
• Advisor Reports: Generate comprehensive class performance summaries.

Slide 10: Governance & Administration
[Principal / HOD]
• Full Institutional Audit: Comprehensive view of department progress.
• Risk Intelligence: Proactive identification of "At-Risk" students.
• Regional Performance: Faculty and departmental comparisons.
[Admin]
• Bulk Management: Large-scale CSV imports for users and subjects.
• System Lifecycle: Automated Semester Transitions and Promotions.

Slide 11: Workflow
• Student Solution Upload -> Evaluation Engine Trigger.
• Theory Path: Image -> OCR -> AI Comparison -> Score.
• Coding Path: Python -> Execution -> Test Cases -> Score.
• Result Storage -> Dashboard Update -> User Notification.

Slide 12: Requirements
• Software: React, Node.js, Express, MongoDB, Python (FastAPI), OCR, LLM.
• Hardware: 2.5GHz Quad-core+, 8GB-16GB RAM, 10GB Storage.

Slide 13: Conclusion
• Modernizing academic excellence with data-driven AI solutions.
• Reducing faculty load while improving student outcomes.
